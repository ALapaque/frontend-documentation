`vue-i18n` est la bibliothèque de référence pour traduire une application Vue. En Vue 3, tu l'emploies en **mode Composition** (`legacy: false`) : les traductions deviennent une fonction `t()` réactive, et changer de locale rerend l'interface sans rechargement. Le principe reste universel : un message est un **patron** paramétré, jamais une chaîne assemblée à coups de `+`.

## Installer et lire un message

`createI18n` configure la locale active et les catalogues, puis se branche sur l'app.

```js
import { createI18n } from "vue-i18n";

const i18n = createI18n({
  legacy: false,          // mode Composition
  locale: "fr",
  fallbackLocale: "en",
  messages: {
    fr: { home: { title: "Bienvenue" } },
    en: { home: { title: "Welcome" } },
  },
});
app.use(i18n);
```

Dans un composant `<script setup>`, tu récupères `t` via `useI18n` ; dans le template, `$t` est disponible directement.

```vue
<script setup>
import { useI18n } from "vue-i18n";
const { t, locale } = useI18n();
</script>

<template>
  <h1>{{ t("home.title") }}</h1>
  <button @click="locale = 'en'">English</button>
</template>
```

Changer `locale` rerend tout ce qui dépend de `t` : c'est réactif, pas besoin de rechargement.

## Pluralisation et interpolation

Le ternaire `n === 1 ? "voiture" : "voitures"` casse hors français/anglais. `vue-i18n` exprime les formes plurielles séparées par `|`, et choisit la bonne selon le nombre.

```js
messages.fr.cars = "pas de voiture | une voiture | {count} voitures";
```

```vue
<p>{{ t("cars", count, { count }) }}</p>
```

Pour les langues à plus de deux formes (russe, polonais), `vue-i18n` permet de définir une **règle de pluralisation** par locale qui mappe le nombre vers le bon index. L'interpolation nommée (`{count}`) garde les variables hors du texte traduit.

:::callout{type="warn"}
**Ne concatène pas des morceaux traduits.** `t("tu_as") + n + t("messages")` est ingérable : l'ordre des mots et l'accord changent d'une langue à l'autre. Mets la phrase entière dans une seule clé avec interpolation et pluralisation — `vue-i18n` recolle dans le bon ordre pour chaque langue.
:::

## Formater nombres et dates

`vue-i18n` expose des helpers qui s'appuient sur `Intl` : tu déclares des formats nommés par locale, puis tu les appliques.

```js
const i18n = createI18n({
  legacy: false,
  numberFormats: {
    fr: { devise: { style: "currency", currency: "EUR" } },
  },
});
```

```vue
<p>{{ n(1234.5, "devise") }}</p>   <!-- 1 234,50 € -->
```

Sous le capot, c'est `Intl.NumberFormat`/`Intl.DateTimeFormat` — voir le module web sur `Intl` pour les règles de devise et de fuseau.

## Charger les locales paresseusement

Embarquer toutes les langues gonfle le bundle pour rien. Charge chaque catalogue à la demande, puis injecte-le avec `setLocaleMessage`.

```js
async function changerLocale(code) {
  if (!i18n.global.availableLocales.includes(code)) {
    const msgs = await import(`./locales/${code}.json`);
    i18n.global.setLocaleMessage(code, msgs.default);
  }
  i18n.global.locale.value = code;
}
```

Le bundler isole chaque locale dans son propre chunk, téléchargé uniquement à la première bascule vers cette langue.

## RTL : miroiter la mise en page

Pour l'arabe ou l'hébreu, applique `dir` sur la racine et laisse le CSS suivre.

```js
watch(locale, (code) => {
  document.documentElement.dir = code === "ar" ? "rtl" : "ltr";
});
```

Emploie les **propriétés logiques** CSS (`margin-inline-start`, `padding-inline-end`) plutôt que `left`/`right` : elles se miroitent automatiquement selon `dir`.
