:::callout{type="info"}
2026 fait franchir à CSS un cap **logique** : après les variables et `calc()`,
voici les **conditions** (`if()`), les **fonctions** (`@function`) et les
**comptages de fratrie** (`sibling-index()`). En parallèle, deux gros morceaux des
années précédentes — anchor positioning et scroll-driven animations — atteignent
**Baseline** (support multi-navigateurs). Les nouveautés sont d'abord dans Chromium ;
guette-les derrière `@supports`.
:::

## `if()` : des conditions en ligne

Jusqu'ici, conditionner une valeur imposait une variable intermédiaire ou un
sélecteur dédié. La fonction **`if()`** met la condition dans la valeur elle-même.

```css
.badge {
  /* selon une custom property, sans classe supplémentaire */
  background: if(style(--ton: danger): crimson; else: var(--accent));
}
```

**Pourquoi.** `if()` lit une condition (`style()` sur une custom property, à terme
`media()` et `supports()`) et renvoie une valeur. On évite la multiplication de
classes utilitaires pour de simples bascules : la logique vit là où la valeur est
consommée. À combiner avec parcimonie — une feuille de style n'est pas un langage
de programmation.

## `@function` : des fonctions personnalisées

Les custom properties stockent des valeurs ; **`@function`** stocke un **calcul
paramétré** réutilisable.

```css
@function --fluid(--min, --max) {
  result: clamp(var(--min), 4vw, var(--max));
}

h1 { font-size: --fluid(1.5rem, 3rem); }
```

**Pourquoi.** Avant, une typo fluide imposait de recopier un `clamp()` partout ou
de générer le CSS via Sass. `@function` rend le motif natif, avec des arguments
typés et une valeur de repli — la logique reste dans le navigateur, sans étape de
build.

## `sibling-index()` et `sibling-count()`

Styler le n-ième élément demandait des `--i: 3` posés à la main. Ces fonctions
exposent la **position dans la fratrie** directement.

```css
li {
  /* décalage progressif sans variable manuelle par item */
  transition-delay: calc(sibling-index() * 60ms);
  opacity: calc(1 - sibling-index() / sibling-count());
}
```

**Pourquoi.** Les animations en cascade (stagger) et les dégradés positionnels
n'exigent plus de générer un index côté JS ou template. Le navigateur connaît la
position ; CSS la lit.

## Masonry natif

Le layout en maçonnerie (colonnes à hauteurs décalées, façon Pinterest) imposait
une bibliothèque JS. Il arrive **dans Grid**.

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-template-rows: masonry;
}
```

:::callout{type="warn"}
La syntaxe exacte fait encore débat (`grid-template-rows: masonry` contre une
propriété `item-flow` dédiée) entre éditeurs de navigateurs. Le **concept** est
acquis et s'expédie ; le **mot-clé** peut bouger. Isole-le derrière `@supports`
plutôt que de le parsemer dans tes feuilles.
:::

## Ce qui atteint Baseline

Deux features déjà étudiées passent en support multi-navigateurs en 2026 : voir
`anchor-positioning` (ancrer un élément à un autre, sans JS de positionnement) et
`scroll-animations` (animations pilotées par le défilement). L'enjeu n'est plus
« est-ce supporté » mais « puis-je retirer le repli » — surveille Baseline.
